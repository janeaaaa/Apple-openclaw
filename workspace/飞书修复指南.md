# 飞书消息路由问题修复指南

## 问题描述
飞书发送消息后，回复被路由到独立子会话（`agent:main:feishu:direct:xxx`）而不是主会话，导致无法收到回复。

## 根因分析
- openclaw.json 中 `session.dmScope` 配置为 `per-channel-peer`
- 这会为每个飞书用户创建独立的会话，导致消息无法汇聚到主会话

## 修复步骤

### 1. 备份原配置
```powershell
# 备份 openclaw.json
Copy-Item "C:\Users\Administrator\.openclaw\openclaw.json" "C:\Users\Administrator\.openclaw\openclaw.json.bak"
```

### 2. 修改 session.dmScope
将 `per-channel-peer` 改为 `main`

```json
// 修改前
"session": {
  "dmScope": "per-channel-peer"
}

// 修改后
"session": {
  "dmScope": "main"
}
```

### 3. 重启 Gateway
```powershell
openclaw gateway restart
```

### 4. 验证修复
- 飞书发送测试消息
- 检查日志确认消息路由到 `agent:main:main`

---

## 修复代码

### 方法1: 直接编辑 openclaw.json

找到以下配置：
```json
"session": {
  "dmScope": "per-channel-peer"
}
```

修改为：
```json
"session": {
  "dmScope": "main"
}
```

### 方法2: 使用 PowerShell 命令

```powershell
# 读取配置文件
$config = Get-Content "C:\Users\Administrator\.openclaw\openclaw.json" | ConvertFrom-Json

# 修改 session.dmScope
$config.session.dmScope = "main"

# 保存配置
$config | ConvertTo-Json -Depth 10 | Set-Content "C:\Users\Administrator\.openclaw\openclaw.json"
```

### 方法3: 使用 edit 工具

```javascript
// 编辑 oldText
"session": {
  "dmScope": "per-channel-peer"
}

// 编辑 newText
"session": {
  "dmScope": "main"
}
```

---

## dmScope 可选值说明

| 值 | 说明 |
|---|---|
| main | 所有消息路由到主会话 |
| per-peer | 每个用户一个会话 |
| per-channel-peer | 每个频道+用户一个会话（默认） |
| per-account-channel-peer | 每个账号+频道+用户一个会话 |

推荐使用 `main` 确保所有飞书消息汇聚到主会话。
