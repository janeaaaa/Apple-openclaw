# 团队工作流设计方案

## 一、团队概述

### 1.1 团队成员
| 智能体 | 名称 | 职能 | 模型 | 直接汇报对象 |
|--------|------|------|------|--------------|
| 主管 | 小苹果🍎 | 任务分配、进度汇报、对外沟通 | MiniMax-M2.5 | 静静 |
| 内容创作 | 小帅📝 | 公众号写作、SEO优化、热点追踪 | MiniMax-M2.5 | 小苹果 |
| 视觉设计 | 小美🎨 | 图像生成、图片处理 | MiniMax-M2.5 | 小苹果 |
| 客服接待 | Zero💬 | 社群运营、回复消息 | MiniMax-M2.5 | 小苹果 |

### 1.2 协作模式
- 静静 → 小苹果 → 下属智能体
- 信息流：单向流动，静静定目标，小苹果执行和汇报

## 二、每日任务流程

### 2.1 早间（7:00-9:00）
| 时间 | 任务 | 执行者 | 产出 |
|------|------|--------|------|
| 7:00 | 抓取AI热点 | 小苹果 | 热榜数据 |
| 7:00-7:30 | 分析热点，筛选选题 | 小苹果 | 选题建议 |
| 7:30-8:00 | 分配任务给小帅 | 小苹果 | 任务派发 |
| 8:00-9:00 | 向静静汇报早间工作 | 小苹果 | 汇报消息 |

### 2.2 上午（9:00-12:00）
| 时间 | 任务 | 执行者 | 产出 |
|------|------|--------|------|
| 9:00-10:30 | 公众号写作 | 小帅 | 初稿 |
| 10:30-11:00 | 进度检查 | 小苹果 | 状态更新 |
| 11:00-12:00 | 排版优化 | 小帅 | 定稿 |

### 2.3 中午（12:00-14:00）
- 午间检查：小苹果汇报上午进度

### 2.4 下午（14:00-18:00）
| 时间 | 任务 | 执行者 | 产出 |
|------|------|--------|------|
| 14:00 | 日常任务检查 | 小苹果 | 状态更新 |
| 14:00-17:00 | 设计/客服任务（如有） | 小美/Zero | 完成产出 |
| 17:00-18:00 | 下班汇报准备 | 小苹果 | 汇总报告 |

### 2.5 下班后（18:00-20:00）
- 18:00 下班汇报
- 20:00 日常任务检查
- 如有遗留任务，优先完成

## 三、任务分配规则

### 3.1 公众号写作任务
```
静静（目标） → 小苹果（拆解） → 小帅（执行） → 小苹果（检查） → 静静（审核）
```

**具体流程**：
1. 静静给出目标（方向、浏览量目标等）
2. 小苹果分析热点，确定选题
3. 小帅创作初稿
4. 小帅使用wechat-article-publisher发布
5. 小帅汇报结果

### 3.2 设计任务
```
静静（需求） → 小苹果（派发） → 小美（执行） → 小苹果（验收）
```

### 3.3 客服任务
```
外部消息 → 小苹果（分配） → Zero（处理） → 小苹果（归档）
```

## 四、汇报机制

### 4.1 定时汇报
| 时间 | 内容 | 汇报者 |
|------|------|--------|
| 7:00 | 当日计划、热点 | 小苹果 |
| 12:00 | 上午进度 | 小苹果 |
| 18:00 | 全天汇总 | 小苹果 |
| 20:00 | 遗留任务检查 | 小苹果 |

### 4.2 实时汇报（遇以下情况立即汇报）
- 任务完成
- 任务失败
- 遇到问题需要协助
- 静静有新安排

## 五、任务优先级

### 5.1 优先级定义
| 优先级 | 定义 | 示例 |
|--------|------|------|
| P0 | 紧急重要，静待完成 | 静静直接安排的任务 |
| P1 | 当天必须完成 | 每日必做任务 |
| P2 | 今日尽量完成 | 计划内任务 |
| P3 | 可延后 | 优化类任务 |

### 5.2 判定规则
1. 静静直接安排 → P0
2. 每日必做任务（热点、公众号）→ P1
3. 计划内任务 → P2
4. 优化类任务 → P3

## 六、质量控制

### 6.1 公众号质量标准
- 标题吸引人
- 内容有深度
- 排版统一（standard模板）
- 封面符合风格

### 6.2 检查清单
- [ ] 标题是否符合要求
- [ ] 内容是否有价值
- [ ] 排版是否统一
- [ ] 封面是否合适

## 七、风险管理

### 7.1 常见风险
| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 热点抓取失败 | 无法选题 | 使用tophub.today备选 |
| 公众号发布失败 | 无法更新 | 保留草稿，手动发布 |
| API超时 | 任务中断 | 重试或跳过 |
| 静静不在 | 无法确认 | 等待确认后继续 |

### 7.2 升级流程
```
一般问题 → 小苹果处理
重要问题 → 汇报静静
```
