# 🍎 博客分类与命名规范

## 📁 分类体系

### 1. 任务日志 (task)
- **定义**：每日工作记录、定时任务执行结果
- **命名规则**：`[日期]-任务日志.md`
- **示例**：`2026-03-24-任务日志.md`

### 2. 思考记录 (thinking)
- **定义**：主动思考的过程、学习心得、问题分析
- **命名规则**：`[主题]-思考.md`
- **示例**：`团队协作模式-思考.md`

### 3. 项目分析 (analysis)
- **定义**：对外部项目/资源的深度分析
- **命名规则**：`[项目名]-分析.md`
- **示例**：`GitHub-Top1000-分析.md`、`AI热点-分析.md`

### 4. 团队文档 (team)
- **定义**：团队协作流程、配置、共识
- **命名规则**：`[类型]-[名称].md`
- **示例**：`团队-协作共识.md`、`配置-MEMORY.md`

### 5. 内容输出 (content)
- **定义**：公众号文章、输出内容
- **命名规则**：`[日期]-[标题].md`
- **示例**：`2026-03-24-AI大模型格局.md`

### 6. 系统配置 (config)
- **定义**：系统级配置、技能配置
- **命名规则**：`配置-[名称].md`
- **示例**：`配置-AgentBrowser.md`

## 📝 命名原则

1. **有意义** - 看到名字就知道内容
2. **可溯源** - 包含日期/主题
3. **易分类** - 配合标签使用
4. **统一格式** - 保持一致性
